<%%>是指令标识。

基本上就是说，使用<%%>指令标识，里面的代码块就会被认为是指定语言的代码块

表示不是直接由客户端接受的，而是由服务器端解释后，传给客户端接受的。
具体是什么语言，在html文件的开头有标签解释。(<%@ language = java …%>)

include指令
通过这个指令，一个jsp文件可以包含另一个jsp文件；
但这种包含是静态包含，相当于将被包含的jsp文件全部拷贝过来，不会执行被包含文件的jsp代码
提高代码的重用性（例5.11）

<%! ……%>声明标识
用于声明变量或方法，在本页面全局可用

<%— ……—%>隐藏注释
这样的注释方式，在客户端看source file时是看不到注释信息的

<!—<%=newDate()%>—>动态注释
html的注释，对java命令标识不起作用，所以<%%>内的代码还是会运行，
故在看源代码注释的时候，还是会显示当前的时间

动作标识
这也是将所include的页面包含进来。
但这个所包含的即可以是静态文件，也可以是动态文件。
意思就是说在文件执行过程中，被包含的文件会被服务器编译并执行。

（一）Spring-Boot环境配置与HelloWorld

2016-10-30
java学习Spring-Boot
java, java-web

初视Spring-Boot

Spring-MVC框架（Framework）是早有耳闻的，其中对很多抽象概念进行了实现，并且用注解（annotations）就可以方便的使用了。官网是这么概括Spring-mvc框架的：

可以见到，整个框架就是围绕DispatcherServlet来建设的，这本身就是一个servlet，不过他的功能是将不同的request分配到不同的handlers上，再用handler进行处理，所以我们使用Spring-mvc框架，就是封装隐藏了request怎么接受，怎么解包，怎么获取信息，这些底层，繁琐的操作，直接对hanlders这些逻辑进行实现，提高开发效率。

那么我们抽象出来的到底是什么呢？不得不谈谈什么是mvc。

M-V-C架构分别对bs模型中的每个步骤抽象出来的，使得后台的逻辑实现，请求控制，前台的数据展示完全分离，从紧耦合变为松耦合，将整个web开发分为不同的模块开发，各司其职，提高项目的进度和效率。

可以说现在的web开发，不管是使用java，python，c#，以及他们的框架，都是对MVC三层的实现。甚至是手机端的app开发，都是大同小异，只是展现数据的方式不一样，或是服务器与客户端之间发送的包不同，但基本逻辑都是类似的。

MVC架构图：

M
model层是关键的层级，涉及到对抽象业务对象和业务逻辑的具体实现，其实是表示数据库，但并不是包括数据库的全部操作。谨记，这一层是逻辑实现层。
V
view是表示层，可疑对应于web开发中的前端。
C
controller是控制层，对应于web开发中，就是控制request的接受，分发，处理。这里面也包括了对数据库的增删改查操作。
