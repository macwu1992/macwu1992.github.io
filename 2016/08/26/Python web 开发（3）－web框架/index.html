web最主要的功能就是对url进行处理。在python中，url处理主要使用aiohttp类库中的类web。
根据对aiohttp类库的官方说明文档的阅读，对url最基本的处理有以下几步：
1）创建handler函数，接收request对象（url－>request这一步已经由aiohttp完成），并返回web.respond()，返回类型并不限定，亦可是其他类型，但当返回是其他类型时，需要在app = web.Applicetion（middleware=[logger_factory, response_factory]）,将其它类型包装为web.response类型
2）将一个app实例化，app=web.Application()    
3）将handler函数加入app 的路径
4）运行web.run(app)

code：

分清楚几个概念：
1）app 只是运行于web服务器上的程序，并不是web服务器本身，web服务器本身是交给aiohttp实现的
2）app 这个web程序所要实现的功能由添加handler函数及其想对路径来实现

在将Application实例化时，加入参数：middleware，列表中的 response_factory是用于将index（）等handlers的返回类型处理为web.Response()类型，因为在诸如index（）的handlers，在相应“GET”请求时，其实返回的是需要middleware处理的类型，并不是web.Response()，在想要在index（）返回一个非web.Response()，如模板页面(test.html)的话，必须经过response_factory的处理，成为一个web.Response()，才能被浏览器解析
