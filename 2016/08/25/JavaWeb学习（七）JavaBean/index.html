java, java-web

JavaBean其实是对各种事物的编程抽象，它自身是一个class，但它的意义并不是提供方法或是接口，而是对一个真实事物的模拟
例如，书籍类Book，含有书名、作者、ISBN号、版次，等等信息，这就是对真实事物的抽象后编程实现。

但有这么四点是一定要遵循的：

1、所有属性为private
2、提供默认构造方法
3、提供getter和setter
4、实现serializable接口（not must but suggested）

但就如第一块源码所示，定义好了新建好了Email类，是一定要继承Serializable的。
那么问题来了，既然不是必须的，为什么javabean一定要implements Serializeable呢？
首先，这里是oracle官方api_doc给出的Serializebale的说明：
public interface Serializable
Serializability of a class is enabled by the class implementing the java.io.Serializable interface. Classes that do not implement this interface will not have any of their state serialized or deserialized. All subtypes of a serializable class are themselves serializable. The serialization interface has no methods or fields and serves only to identify the semantics of being serializable.

这里说了，实现Serializable接口是向外界表明，此类是可序列化的，是一个标识，并且，该类的子类也都是可序列化的。
但Serializabal本身没有实现方法，所以implements Serializable仅仅是identify ，表明此类是可序列化的。

但真正的序列化和反序列化工作并不是由Serializable来做。具体的序列化工作是有方法来做的：
private void writeObject(java.io.ObjectOutputStream out)
throws IOException
private void readObject(java.io.ObjectInputStream in)
throws IOException, ClassNotFoundException;
private void readObjectNoData()
throws ObjectStreamException;

这里writeObject方法是用来序列化，readObject方法用来反序列化的。
