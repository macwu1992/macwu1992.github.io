利用Hexo建设个人博客

2016-02-14
hexo
git, hexo

　　记得小学的课程作业里，最不喜欢的的就是写日记。一是觉得每天的生活都一样，真不知道流水账有什么好记的，二是觉得写日记很烦，没什么用。现在想想，也许当时是讨厌拿起笔，毕竟我现在用键盘写着东西觉得挺舒服的。一路从学生时代走过来，到现在的研究生阶段，越走越觉得记录是个好习惯，不论是对于学习、工作还是生活，对于我这个既懒惰又有归纳癖的人来说，把一些事情，做事的过程记录下来，乃至每一个步骤，都可以让宝贵的脑容量轻松一些。（也许是缘于小时候百看不厌的鲁滨逊漂流记）😀
　　那么就从搭建我自己的博客开始吧。。。

Hexo on Git

　　Hexo是一款轻量级的博客生成系统（我自己的概念），既可以在本地预览效果，也可以部署到Github这种真人在线同性交友平台上，无异于一个小型的网站日记本，啥都能记，还能定制，想想都有点小激动呢。

官方网站：hexo

　　那么搭建博客的主要材料就是Hexo 加上 Github了。
（使用Gitcafe也可，但亲测Github较为稳定，并能广交全世界同性好友，赶紧去注册吧～）

Github

Gitcafe
　　git的安装祥见github的官网吧。

Hexo的安装

　　1.npm install hexo-cli -g //使用npm安裝hexo客戶端

　　2.hexo init blog //使用“blog“文件初始化

　　3.cd blog//

　　4.npm install

　　5.hexo server //启动本地hexo服务
　　短短五行代码就能搞定hexo的安装，前提是npm这个linux必备的工具要安装好。npm(linux 下的工具多得很，可玩行也很高，目前正在看《鸟哥的linux私房菜》读书笔记也过段时间就放出吧！)

Hexo的部署

　　在Hexo安装之后，首先执行执行hexo g即generate，生成博客页面文件。这时只要执行hexo s就可以启动hexo的本地服务器http://0.0.0.0:4000，就可以访问到自己的本地博客页面，注意，这里是本地服务，也就是外网是访问不到的。想要外网访问，还要进行进一步的设置，就可以将hexo部署在Github上。
　　1.若无Github帐号，则先注册Github帐号，username作为示例帐号。

　　2.建立一个新的repo，命名为username.github.io，这里repo名字为必须的格式，不然无法进行正常访问。

　　3.在hexo文件夹内找到_config.xml，打开之后查找Deployment，在git下：
　　这一步非常重要，需要注意的是Github的地址，以及Git的设定
　　

　　4.cd hexo，进入本地hexo文件夹，执行

　　5.hexo server //启动本地hexo服务
　　此时访问http://username.github.io即可。
　　如能正确访问到博客，hexo的安装与部署已完成，如不能则仔细检查上述步骤。至于主题的更改以及后期的美化待更。
